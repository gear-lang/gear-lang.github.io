Define documentation

#define GEAR_COMPUTE_ARITHMETIC_IN_VM_FOR_NUMERIC_TYPES

If enabled, arithmetic computation will be performed in the VM for integers and floats.

In Gear, arithmetic computation is performed by sending a message to an object. This is how the integer and float types perform their arithmetic. The advantage of this design is their behavior can be changed at runtime. The disadvantage is there is a performance overhead.

By defining this macro the VM is instructed to perform the computation itself instead of sending a message. This improves performance at the cost of flexibility, but in practice most applications do not need this flexibility.

An example of an application that might want this flexibility is scientific applications. By overriding the default behavior an application could implement "big number" arithmetic.

#define GEAR_API

Shared library export macro.

The macro GEAR_BUILD_AS_DLL should be defined when building Gear as a shared object on Windows. This is already done by the CMake script. If a custom build system is used, then this will also need to be done by said system when building Gear as a shared object on Windows.
