Typedef documentation

typedef unsigned long gear_variable

A native variable.

This is an abstract handle representing a Gear value. It should not be copied, inspected, or assigned directly. See gear_assign for coping values between variables.

While a value is referenced by a variable it cannot be garbage collected.

void gear_return(gear_state* state, gear_variable variable)

Causes execution to leave the current native C subroutine.

Returns The variable containing the return result of the current function.

Remember to release all allocated memory before invoking this function. Any allocated memory that is not free'd will leak.

This should only be called within the context of a native C function. If it's called from any other context, then GEAR_INVALID_OPERATION is generated.

If this function is never called, then the native C function defaults to returning nil.

void add_two_integers(gear_state *s)
{
    gear_int x, y;
    gear_variable result;

    // Initialize the local variable.
    gear_init_local(s, &result);

    // Grab the values of the two arguments.
    gear_argvi(s, 0, &x);
    gear_argvi(s, 1, &y);

    // Sum the arguments and store the result in the local variable.
    gear_assign_new_int(s, x + y, result);

    // Return the value of the variable.
    gear_return(s, result);
}

gear_int gear_to_int(gear_state* state, gear_variable variable)

Converts the value referenced by the given variable to an integer.

Parameters
state in The runtime context.
variable in A variable to a value.
Returns The integer value.

If the value is an integer, then it is returned as-is. If the value is a float, then its factional is discarded and returned an as integer. If the value is a boolean, then 1 is returned if it's true and 0 if it's false. If the value is anything else, then 0 is returned.

Take note that this does not send the toInt message. Consumers must call toInt themselves using gear_call.
