5. Functions

Embedding

You might like to give your users the ability to extend or modify your application. One way to do this is by using a shared object. Unfortunatly shared objects have a few problems:

  • They must be recompiled for every platform.
  • They have unrestricted access to the system.
  • They share the applications memory space so a crash will bring down the whole application

Gear solves these problems by executing code in a sandboxed environment. This means all code is run through a virtual machine and garbage collected). This protects your applications memory and insulates it from crashes.

Users will not have unrestricted access to the system because the host can choose what functionality to expose to scripts.
